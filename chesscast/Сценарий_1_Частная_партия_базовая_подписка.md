# Сценарий 1: Проведение частной партии пользователем с базовой подпиской

| №   | Действие специалиста                                                         | Результат                                                                                          | Время, сек. |
| --- | ---------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- | ----------- |
| 1   | Открывает приложение в браузере                                              | Отображается главная страница приложения                                                           | 2           |
| 2   | Нажимает кнопку "Войти"                                                      | Открывается страница входа                                                                         | 1           |
| 3   | Вводит email и пароль, нажимает "Войти"                                      | Происходит аутентификация, пользователь перенаправляется в личный кабинет                          | 3           |
| 4   | Нажимает кнопку "Создать игру"                                               | Открывается модальное окно создания игры                                                           | 1           |
| 5   | Заполняет форму создания игры (название, описание), нажимает "Создать"       | Создается игра, генерируется уникальный токен игры, пользователь перенаправляется на страницу игры | 4           |
| 6   | Устанавливает телефон/камеру на штатив, направляет на пустую шахматную доску | Камера готова к калибровке                                                                         | 10          |
| 7   | Нажимает кнопку "Начать калибровку"                                          | Открывается интерфейс калибровки с предпросмотром камеры                                           | 2           |
| 8   | Нажимает кнопку "Сделать снимок" для калибровки                              | Кадр отправляется на сервер для обработки                                                          | 1           |
| 9   | Ожидает обработку калибровки                                                 | Сервер обрабатывает изображение, определяет границы доски и клетки, проверяет пустоту доски        | 5           |
| 10  | Получает результат калибровки                                                | Если доска пуста - калибровка успешна, маппинг сохранен. Если нет - отображается ошибка            | 2           |
| 11  | Расставляет фигуры в начальную позицию на доске                              | Доска готова к игре                                                                                | 30          |
| 12  | Нажимает кнопку "Начать игру"                                                | Система проверяет подписку пользователя                                                            | 2           |
| 13  | Система определяет базовую подписку, показывает рекламный блок               | Отображается рекламное объявление (видео/баннер)                                                   | 15          |
| 14  | Просматривает рекламу до конца                                               | Реклама завершена, активируется доступ к игре                                                      | 15          |
| 15  | Нажимает кнопку "Продолжить" после рекламы                                   | Устанавливается WebSocket соединение с сервером                                                    | 2           |
| 16  | Система отправляет событие `start-stream`                                    | Сервер запускает процесс обработки потока, инициализирует трекер фигур                             | 3           |
| 17  | Получает подтверждение `stream-started`                                      | Начинается передача кадров с камеры на сервер (2 FPS)                                              | 1           |
| 18  | Начинает играть, делая ходы на доске                                         | Система отслеживает перемещения фигур в реальном времени                                           | -           |
| 19  | Система детектирует ход                                                      | Ход отображается на виртуальной доске, запускается анализ позиции через Stockfish                  | 2           |
| 20  | Получает оценку позиции и лучший ход                                         | Отображается оценка позиции, глубина анализа, рекомендуемый ход                                    | 1           |
| 21  | Продолжает игру, делая следующие ходы                                        | Процесс повторяется для каждого хода                                                               | -           |
| 22  | Завершает партию, нажимает "Завершить игру"                                  | Останавливается поток обработки, закрывается WebSocket соединение                                  | 2           |
| 23  | Система сохраняет партию                                                     | Партия сохранена в истории игр пользователя                                                        | 3           |

**Общее время активных действий: ~95 секунд (без учета времени игры)**
